# Task Contract: PR19.1 — Operational runbooks & incident response

## Outcome
Operators have keyed runbooks and an incident-response playbook for the common
ProcessEngine failure modes.

## Scope
- In: `docs/ops/runbooks/` (stuck process, poison message, outbox backlog, lease
  storm, broker/DB outage, wrong-version pinning) + an incident playbook.
- Out: org-specific escalation policy.

## Affected module
Ops docs; keyed to PR10.1 metrics.

## Acceptance — frozen
- [ ] A runbook per listed failure mode with detection signal (metric from
      PR10.1), diagnosis query (SQL against ProcessEngine tables), and remediation.
- [ ] An incident-response playbook ties signals → runbooks → escalation.
- [ ] Each runbook's diagnosis query is validated against the live schema.

## Required tests
Validate each SQL diagnosis query against the deployed schema.

## Dependencies
PR10.1. **Priority** P1 · **Size** M · **Blocks stable** no.

## Docs
`docs/ops/runbooks/`.
