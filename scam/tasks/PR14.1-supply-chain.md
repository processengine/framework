# Task Contract: PR14.1 — SBOM, dependency scanning & npm provenance

## Outcome
Every published package ships with an SBOM and provenance, and CI blocks on
high-severity dependency vulnerabilities.

## Scope
- In: CI workflow, publish workflow (`--provenance`), CycloneDX SBOM generation,
  OSV/`npm audit` gate; `docs/security/SUPPLY_CHAIN.md`.
- Out: signing infra beyond npm provenance/Sigstore defaults.

## Affected module
Repo CI/release; all three packages.

## Acceptance — frozen
- [ ] A CycloneDX SBOM is produced per package build artifact.
- [ ] CI fails on a high/critical advisory (proven with a temporary pinned bad dep
      in a test branch, then reverted).
- [ ] `npm publish --provenance` is used; published packages show provenance on
      the registry.

## Required tests
CI job assertions; a provenance verification step post-publish.

## Dependencies
PR15.1 (CI), PR16.1 (release). **Priority** P0 · **Size** M · **Blocks stable** yes.

## Docs
`docs/security/SUPPLY_CHAIN.md`.

## Stop conditions
None expected.
