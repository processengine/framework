# Task Contract: PR13.1 — Backpressure, bounded concurrency & graceful degradation

## Outcome
The worker bounds concurrency and sheds load predictably under overload/broker
slowness (observed: a single publish to a down broker blocks 3–16s), with a
documented graceful-shutdown drain contract.

## Scope
- In: `conductor` worker loop (bounded in-flight dispatch, tunable batch/lease),
  `transport-kafka` publish timeout tightening; `docs/ops/LIMITS.md`.
- Out: autoscaling policy.

## Affected module
`conductor/src/conductor.ts` (worker), `transport-kafka`.

## Acceptance — frozen
- [ ] Configurable bound on concurrent in-flight dispatches; documented default
      resource requests/limits.
- [ ] Under a broker outage, outbox retry latency is bounded and documented; no
      unbounded memory growth (asserted in an overload test).
- [ ] Graceful shutdown drains or safely releases claims (no lost/duplicated
      effect) — asserted.

## Required tests
Overload unit/integration test; shutdown-drain test.

## Dependencies
PR10.1 (metrics to observe), PR12.1 (load). **Priority** P1 · **Size** M ·
**Blocks stable** no.
