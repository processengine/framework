# Task Contract: PR10.1 — OpenTelemetry, metrics & structured logs

## Outcome
The conductor emits OpenTelemetry spans, a documented metric set, and structured
logs via pluggable hooks (no hard OTel dependency), so operators can observe
dispatch, completion, timeout, lease and outbox behavior in production.

## Scope
- In: `conductor/src/conductor.ts`, `kernel.ts` (instrumentation hooks); a small
  telemetry port in `spi.ts`; `docs/observability/{OTEL,METRICS}.md`.
- Out: shipping an exporter implementation; dashboards (runbooks PR19.1).

## Affected module
`@processengine/conductor` (+ optional adapter in host-adapter).

## Acceptance — frozen
- [ ] Spans exist for start / dispatch / completion / transition with process and
      operation identifiers as attributes.
- [ ] Documented metrics emitted: outbox depth, dispatch attempts, dispatch
      failures, completion timeouts, lease reclaims, completion latency.
- [ ] Zero-config default is a no-op (no dependency, no output).
- [ ] A test wiring a fake exporter observes the spans + metric set.
- [ ] `npm --prefix processengine run check` exit 0.

## Required tests
Span/metric emission unit tests with an in-memory exporter.

## Dependencies
PR1.1 (public surface for the telemetry port). **Priority** P0 · **Size** L ·
**Blocks stable** yes.

## Docs
`docs/observability/OTEL.md`, `docs/observability/METRICS.md`.

## Stop conditions
No PII/domain payload in spans/metrics/logs by default.
