# Task Contract: PR11.1 — Health/readiness contract & diagnostic API

## Outcome
A package-level health/readiness contract (storage reachable, transport connected)
and a read-only process/operation/outbox diagnostic API that hosts can expose.

## Scope
- In: `conductor` health/readiness surface + a read-only inspection API;
  host-adapter guidance; `docs/ops/HEALTH.md`.
- Out: metrics (PR10.1), auth for the diagnostic API (host concern).

## Affected module
`@processengine/conductor`, host-adapter.

## Acceptance — frozen
- [ ] Readiness reports NOT-ready on storage or transport loss and recovers when
      restored (asserted in a test with an injected failing SPI).
- [ ] A read-only API returns process, operation, inbox and outbox records for an
      instance without mutating state.

## Required tests
Readiness transition tests with fault-injected SPI; inspection API contract test.

## Dependencies
None. **Priority** P1 · **Size** M · **Blocks stable** no.

## Docs
